📤 Document Upload Flow

From Browser to S3: Complete Pipeline with Validation, Extraction & Indexing

⏱️ Total Time: 5-10 seconds per document
1
User Selects File (Frontend)
User drags and drops a PDF file or clicks to browse. Frontend validates file before upload.
const handleUpload = async (file: File) => { // Client-side validation if (file.size > 10485760) { throw new Error("File too large (max 10MB)"); } const formData = new FormData(); formData.append("file", file); formData.append("department", "finance"); await fetch("/upload", { method: "POST", body: formData }); };
2
Backend Validation (FastAPI)
Server validates file type, size, user permissions, and file integrity.
File Type .pdf, .docx, .doc, .txt, .md only
File Size Maximum 10MB (HTTP 413 if exceeded)
User Role Must be admin or uploader (HTTP 403)
@app.post("/upload") async def upload_document( file: UploadFile, department: str, current_user: dict = Depends(get_current_user) ): # Validate file type if not file.filename.endswith((".pdf", ".docx", ".txt")): raise HTTPException(415, "Unsupported file type") # Validate file size content = await file.read() if len(content) > 10485760: raise HTTPException(413, "File too large") # Validate user role if current_user["role"] == "viewer": raise HTTPException(403, "Viewers cannot upload")
3
Text Extraction
Extract text from PDF (PyPDF2), DOCX (python-docx), or TXT files. Returns HTTP 422 if extraction fails.
def extract_text(file_bytes: bytes, filename: str) -> str: if filename.endswith(".pdf"): import PyPDF2 reader = PyPDF2.PdfReader(io.BytesIO(file_bytes)) text = "".join([page.extract_text() for page in reader.pages]) elif filename.endswith(".docx"): import docx doc = docx.Document(io.BytesIO(file_bytes)) text = "\n".join([para.text for para in doc.paragraphs]) else: # .txt text = file_bytes.decode("utf-8") if not text.strip(): raise HTTPException(422, "Could not extract text") return text
4
Text Chunking
Split text into overlapping chunks for better context preservation during retrieval.
500
Characters per chunk
50
Character overlap
~40
Chunks per 10-page PDF
def chunk_text(text: str, chunk_size=500, overlap=50) -> List[str]: chunks = [] start = 0 while start < len(text): end = start + chunk_size chunk = text[start:end] if chunk.strip(): chunks.append(chunk) start += (chunk_size - overlap) return chunks # Example output: # ["Our vacation policy allows...", "...allows 15 days per year...", ...]
5
Generate Embeddings
Convert each text chunk into a 384-dimensional vector using sentence-transformers model.
from sentence_transformers import SentenceTransformer model = SentenceTransformer("all-MiniLM-L6-v2") # 384 dimensions def generate_embeddings(chunks: List[str]) -> List[List[float]]: embeddings = model.encode(chunks) return embeddings.tolist() # Example output: # [[0.123, -0.456, 0.789, ...], [0.234, -0.567, 0.890, ...], ...] # Each vector has 384 dimensions
6
Store Vectors (pgvector / Pinecone)
Store embeddings in vector database with metadata for fast similarity search.
# PostgreSQL + pgvector INSERT INTO embeddings ( id, document_id, chunk_index, chunk_text, embedding, metadata ) VALUES ( 'emb_001', 'doc_123', 0, 'Our vacation policy allows...', '[0.123, -0.456, ...]'::vector(384), '{"company_id": "demo-company", "department": "hr"}'::jsonb ); # Create index for fast vector search CREATE INDEX ON embeddings USING ivfflat (embedding vector_cosine_ops);
7
Upload to S3
Store original document in S3 with versioning and encryption enabled.
import boto3 s3 = boto3.client("s3") s3_key = f"companies/{company_id}/documents/{doc_id}/{filename}" s3.put_object( Bucket="your-company-documents", Key=s3_key, Body=file_bytes, Metadata={ "company_id": company_id, "uploaded_by": user_id, "department": department }, ServerSideEncryption="AES256" ) # S3 path: s3://your-company-documents/companies/demo-company/documents/doc_123/policy.pdf
8
Save Metadata to PostgreSQL
Store document metadata in relational database for querying and management.
INSERT INTO documents ( id, company_id, uploaded_by, filename, department, s3_key, file_size, chunk_count, status, created_at ) VALUES ( 'doc_123', 'demo-company', 'user_456', 'vacation_policy.pdf', 'hr', 'companies/demo-company/documents/doc_123/vacation_policy.pdf', 245678, 42, 'active', NOW() );
9
Return Success Response
Send success response to frontend with document ID and metadata.
return { "status": "success", "document_id": "doc_123", "filename": "vacation_policy.pdf", "department": "hr", "chunk_count": 42, "file_size": 245678, "s3_url": "s3://your-company-documents/companies/...", "processing_time": "7.2s", "message": "Document uploaded and indexed successfully" }

📊 Upload Pipeline Summary

Security Validations

  • File type whitelist (.pdf, .docx, .txt only)
  • File size limit (10MB max)
  • User role check (admin/uploader only)
  • JWT token validation
  • Multi-tenant isolation (company_id)
  • Department-level access control

Error Handling

  • HTTP 415: Unsupported file type
  • HTTP 413: File too large
  • HTTP 422: Text extraction failed
  • HTTP 403: Insufficient permissions
  • HTTP 401: Invalid/expired token
  • HTTP 503: Bedrock/S3 unavailable

Performance Metrics

  • Processing time: 5-10 seconds
  • Chunk generation: ~40 chunks per 10-page PDF
  • Embedding dimensions: 384
  • Vector search latency: 50-100ms
  • S3 upload: 1-2 seconds
  • Database writes: <100ms

Storage Details

  • Original file: S3 (versioned, encrypted)
  • Vectors: pgvector (PostgreSQL)
  • Metadata: PostgreSQL (documents table)
  • Audit logs: CloudWatch + local file
  • S3 lifecycle: 30 days → Intelligent-Tiering
  • Database backups: Automated daily
